# AGENTS.md

## Project overview
这是一个古诗背诵 Web 项目，面向移动端使用，支持本地诗库、背诵记录、口语输入解析和推荐。

主要目录如下：
- `src/`: 前端应用代码
- `src/pages/`: 页面组件
- `src/services/`: 数据存储、模型调用、加密等服务层
- `src/domain/`: 领域逻辑
- `api/`: Vercel 服务端接口
- `tests/`: 单元测试
- `docs/`: 项目文档

## Setup
安装依赖：
```bash
npm install
```

启动项目：
```bash
npm run dev
```

本地预览构建产物：
```bash
npm run preview
```

## Validation
提交前必须执行：
- 单元测试：`npm run test`
- build：`npm run build`

当前项目未配置独立 lint 命令，如后续补充，需要把 lint 加入本节。

## Rules
- 处理本项目任何任务前，先阅读 `AGENTS.md`
- 根据任务类型补看 `docs/` 下对应文档：
  - 产品讨论：看 `docs/01-产品现状.md` 和 `docs/05-产品想法.md`
  - 开发前：看 `docs/01-产品现状.md` 和 `docs/02-迭代计划.md`
  - 提交或发布前后：看 `docs/03-开发日志.md` 和 `docs/04-发布清单.md`
- 提交代码后，必须判断是否需要更新 `docs/03-开发日志.md`
- 发布上线后，必须执行 `docs/04-发布清单.md`
- 如果发布依赖用户手动操作，必须明确提醒用户还需要做什么
- 只把已确认事实写入 `docs/01-产品现状.md`
- 还在讨论中的方向只写入 `docs/05-产品想法.md`
- 优先复用已有组件和服务，不要重复造轮子
- 涉及数据结构或接口行为变更时，必须同步更新文档
- 先给最小可运行方案，再做额外重构
- 不能确认的信息不要编，统一标记为“待补充”

## Directory notes
### `src/`
- 保持移动端优先
- 页面逻辑优先放在 `src/pages/`
- 跨页面状态和业务编排集中在 `src/useAppStore.ts`
- 通用数据访问和模型调用放在 `src/services/`
- 纯规则和领域逻辑放在 `src/domain/`

### `api/`
- 当前主要是 Vercel Serverless Function
- 返回结构尽量稳定，错误信息要可定位
- 涉及默认 API 代理行为变更时，要同步检查线上环境变量和文档

### `tests/`
- 当前是最小单元测试集
- 如果修改了领域规则或数据字段，优先补对应测试

### `docs/`
- `01-产品现状.md`：当前已经实现的状态
- `02-迭代计划.md`：已经确认要推进的事项
- `03-开发日志.md`：已完成改动记录
- `04-发布清单.md`：发布前后检查项
- `05-产品想法.md`：还在讨论中的方向
- `decisions/`：重要设计决策记录
