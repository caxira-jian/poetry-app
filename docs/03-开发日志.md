# 开发日志

## 文档用途
- 记录每次迭代做了什么、为什么做、影响了什么。
- 面向未来回看，不代替 git commit。

## 2026-03-17

### 今日推荐排序规则落地
- 内容：实现本地规则驱动的今日推荐排序，新增 `wantToRecite` 和 `lastViewedAt` 字段，首页改为进入即自动推荐，诗库列表改为按统一规则排序，并新增“想背”切换和“建议复习”标记。
- 原因：让首页和诗库共用同一套推荐逻辑，降低对大模型推荐的依赖，并使学习路径更直接。
- 影响：今日推荐改为可预测、可测试的本地规则；诗库成为主要复习入口之一；浏览和背诵行为对排序结果直接生效。
- 相关提交：待补充

### 主导航收缩与规则测试补强
- 内容：底部 Tab 收缩为“今日 / 诗库 / 记录”三项，把模型配置和数据管理移到头部“更多”二级入口；同时补充“想背切换影响排序”和“建议复习前 5 项”测试用例。
- 原因：减少主导航负担，让高频路径更清晰，并把关键规则行为补到自动化测试里。
- 影响：移动端主流程更聚焦，模型/数据类低频操作仍可达；推荐规则回归风险降低。

- 影响：详情页返回体验更接近原生应用。
- 相关提交：`c56976a`
