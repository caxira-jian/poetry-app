# 产品现状

## 文档用途
- 只描述当前已经实现并已确认的产品状态。
- 不写未来设想，不写未确认方案。

## 当前版本状态
- 当前项目已部署到 Vercel。
- 当前为单设备本地数据模式，无账号体系，无云同步。

## 当前页面
- 主导航：今日、诗库、记录。
- 二级入口：模型配置、数据管理。
- 今日：进入页面后自动生成今日推荐。
- 诗库：通过口语输入新增/更新诗词；查看按推荐规则排序的诗库列表；进入诗词详情页；切换“想背”状态。
- 记录：通过口语输入生成背诵记录；查看最近记录。
- 模型配置：切换默认 API / 自定义 API；测试模型连通性；管理主密码。
- 数据管理：初始化种子诗库；JSON 导入导出。

## 当前核心功能

### 诗库管理
- 支持初始化种子诗库。
- 支持口语输入，让大模型解析成结构化动作。
- 入库前会显示解析预览和变更校验。
- 诗库列表默认折叠诗文，支持展开全文。
- 诗库列表按今日推荐排序规则排序。
- 诗库列表标题行右侧提供“想背”切换按钮。
- 诗库前 5 项会展示“建议复习”标记。

### 诗词详情页
- 可从诗库列表进入详情页。
- 顶部有返回按钮。
- 正文按更适合阅读的格式展示。
- 底部有悬浮打卡按钮。

### 背诵打卡
- 点击“背诵打卡”后会弹二次确认。
- 确认后写入一条背诵记录。
- 同时更新背诵次数和最近背诵时间。
- 同一天内再次进入详情页时，按钮显示“今日已打卡”并置灰。

### 浏览次数
- 进入详情页即记 1 次浏览。
- 同时更新最近浏览时间。

### 推荐
- 首页进入后自动生成今日推荐。
- 今日推荐基于本地规则排序，不依赖大模型。
- 首页和诗库共用同一套排序规则。
- 当前只展示今日推荐列表，不再展示新学习建议。

### API 模式
- 默认 API：通过服务端代理调用，访客可直接使用，不暴露 key。
- 自定义 API：用户自己配置 key 和模型，key 在本地加密保存。
- 大模型当前主要用于口语输入解析和模型连通性测试。

- `title`
- `author`
- `content`
- `dynasty?`
- `tags[]`
- `learnIntent`
- `currentStatus`
- `masteryLevel`
- `reciteCount`

### ReciteLog

- `poemId`
- `titleSnapshot`
- `authorSnapshot`
- `recitedAt`
- `status`
- `note?`

## 当前关键交互规则
- 底部主导航只保留 3 项：今日、诗库、记录。
- 模型配置和数据管理通过头部“更多”二级入口进入。
- 首页进入即自动获取最新今日推荐，不做缓存。
- 诗库列表显示：朝代、背诵次数、浏览次数、建议复习标记、最近背诵时间。
- 诗库列表不再显示学习意向。
- `想背` 按钮支持点击开启，再次点击取消。
- 浏览器或系统返回在详情页时优先关闭详情页，回到应用内上一层。
- 默认 API 连通性测试和口语解析通过 `/api/llm` 服务端代理。

## 当前存储方式
- 主存：IndexedDB
- 数据表：`poems`、`recite_logs`、`settings`
- JSON 用于导入导出和迁移

## 当前可观测性
- 服务端代理会记录大模型请求日志。
- 日志包含请求消息、模型返回、token 使用和耗时信息。

## 当前限制
- 没有登录系统。
- 没有多端同步。
- 没有独立路由系统，详情页是应用内页面切换 + history 状态处理。
- 今日推荐规则已本地化，但 `最近浏览时间` 等新增字段的旧数据需要逐步补齐。
